Управление процессами является одной из основных задач операционной системы Linux. Каждая программа, запущенная на компьютере, представляет собой процесс, который выполняет определенные действия и использует системные ресурсы. В данной главе мы рассмотрим основные концепции и инструменты для управления процессами в Linux.

Понятие процесса

Процесс в Linux - это экземпляр исполняющейся программы, который создается операционной системой. Каждый процесс имеет уникальный идентификатор (PID), состояние (запущен, приостановлен, завершен), родителя (процесс, который создал данный процесс) и дочерние процессы (процессы, созданные данным процессом).

Управление процессами с помощью командной строки

Linux предоставляет различные команды для управления процессами с помощью командной строки. Рассмотрим некоторые из них:

  • ps - команда для просмотра списка активных процессов. Она выводит информацию о PID, состоянии процесса, использовании памяти и других параметрах. Например, ps aux позволяет просматривать информацию о всех процессах на системе.
  • top - интерактивная команда для просмотра активных процессов в реальном времени. Она отображает список процессов, которые потребляют наибольшее количество ресурсов (память, процессорное время и прочее).
  • kill - команда для отправки сигналов процессам. Например, kill PID посылает сигнал SIGTERM процессу с указанным идентификатором PID для его завершения. Более жесткий вариант - kill -9 PID отправляет сигнал SIGKILL, который немедленно завершает процесс.
  • nice - команда для управления приоритетом процесса. Она позволяет изменить значение приоритета (от -20 до 19), которое влияет на то, как операционная система распределяет ресурсы. Чем меньше значение, тем выше приоритет.
  • renice - команда для изменения приоритета уже выполняющегося процесса. Например, renice -5 PID снижает приоритет процесса с указанным идентификатором PID.

Управление процессами с помощью утилиты "systemctl"

systemctl – это основная утилита для управления службами и процессами в Linux. Она управляет системными службами и приложениями, которые запускаются в фоновом режиме и выполняются на протяжении всей работы системы. Рассмотрим некоторые команды:

  • systemctl start service - запускает указанную службу или процесс.
  • systemctl stop service - останавливает указанную службу или процесс.
  • systemctl restart service - перезапускает указанную службу или процесс.
  • systemctl status service - выводит информацию о состоянии указанной службы или процесса.
  • systemctl enable service - включает автозапуск указанной службы или процесса при загрузке системы.

Управление процессами с помощью графического интерфейса

В Linux также существуют графические инструменты для управления процессами. Наиболее распространенные из них:

  • "System Monitor" - графическое приложение, которое позволяет просматривать список активных процессов, их используемые ресурсы и другую информацию в удобной форме. Оно также позволяет завершать процессы и изменять их приоритет.
  • "htop" - интерактивный монитор процессов в реальном времени с графическим интерфейсом. Он отображает список процессов, используемые ими ресурсы и позволяет выполнять различные действия с процессами.
